# 五书章节口径说明

> 本文件用于区分不同产物中的章节数字，不替换原文、索引、分章文件或逐章分析结果。遇到异常时保留原始证据，并明确下游分析采用的口径。

## 一、字段定义

同一个篇、卷或集中的“章节数”可能来自不同对象，不能脱离字段名单独比较：

- **索引章节数**：`00_原文索引.md` 根据分割脚本识别出的章节数。
- **分章文件数**：对应 `resources/*_分章/` 目录中的 TXT 文件数。一个文件可能包含多个原文章头，也可能存在重复或变体文件。
- **原文逻辑章节数**：按原文中实际可辨认的章节内容计算；嵌在另一个文件中的后续章也要注明，格式异常的章头不能直接当成正常独立文件。
- **大纲条目数**：逐章大纲实际写出的分析条目数。为了保持可读性，异常章可能合并为一条，并在大纲中说明。
- **CSV 分析条目数**：`五书全量逐章特征数据.csv` 中对应篇/卷/集的分析记录数。它应与大纲条目数一致，不要求与索引或分章文件数相等。

后续报告、任务指派和交接时，必须写成“索引 26 章、分章 26 文件、分析 27 条”这类带字段名的表达，不能只写“26 章”。

## 三、后续维护规则

1. 原文、索引、分章文件、大纲和 CSV 出现差异时，先标明差异属于哪个字段，再说明分析采用的字段。
2. 不为了让数字相等而删除重复/变体文件、拆改原文内容或静默重编号；只有用户明确要求且确认属于标题格式错误时，才可进行最小化标题规范化，并同步更新全部下游产物。
3. 继续分析时默认以对应大纲条目和 CSV 条目作为“分析单元”；需要回到原文时，按本文件的异常说明定位到实际分章文件。
4. 新发现的章节口径差异，应同时补充本文件对应表格和目标大纲的“基本信息/存疑说明”，并在任务日志记录影响范围。
