# 资料查询说明

本文件用于防止 Agent 因查一个章节而加载整份大文件。

## 默认原则

- 大纲、state、附录、CSV 和原文分章文件均采用定点读取。
- 先按标题、章节号、作品名或关键词查询，再决定是否扩大范围。
- 查询结果只作为本次任务证据，不自动变成全库校验。
- 需要扩大范围时，先说明冲突位置和扩大原因。

## 首次接手限制

首次提供仓库地址或只要求了解项目时，不查询业务大文件，不统计文件/章节/数据数量，不执行完整性、正确性、重复、缺失或一致性检查。默认只读取 `AGENTS.md`、`docs/当前接手卡.md`、浅层目录名，以及用途不清时的 README 开头摘要和少量配置/入口文件。

“仔细分析”“全面了解”不自动授权扩大查询；只有用户明确要求核查/审计/统计，或具体修改确实依赖事实核对时，才说明原因并按最低必要范围查询。

## 推荐工具

`tools/query_material.py` 是只读查询工具，支持 Markdown、TXT 和 CSV。

```text
按关键词查大纲或原文：
python3 tools/query_material.py 目标文件 --pattern "第19章" --before 2 --after 18

按标题段落查 state：
python3 tools/query_material.py 目标state --section "核心主线"

取 state 快速视图：
python3 tools/query_material.py 目标state --quick

按作品和全局章节查 CSV：
python3 tools/query_material.py 五书全量逐章特征数据.csv --book "飞剑问道" --chapter 688
```

不使用工具时，也只读取目标文件的局部行段；禁止为了确认一条记录而直接打开全量附录、全量 CSV 或整部原文。

## state 的两种视图

- 快速视图：人物当前状态、核心主线、近章摘要、待确认项。
- 完整视图：伏笔、历史阶段和跨篇连续性资料，仅在相关任务需要时读取。

快速视图按标题即时提取，不另复制一套 state 文件，避免两份状态互相过期。
