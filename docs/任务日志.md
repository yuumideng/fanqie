## 三、技术架构

### 核心方案：单章读取 + 滚动状态档案 + 篇级子Agent并行
详见 `docs/吞噬星空后续大纲生成技术方案.md`。

1. **按章分割**：脚本依据索引把原文切成单章文件，落盘可断点续传
2. **单章读取+滚动state.md**：每篇维护轻量状态档案（人物境界/伏笔/主线/近章摘要），替代全量原文上下文
3. **篇级子Agent并行**：每篇/卷开一个子agent独立处理，多篇同步生成
4. **篇级聚合**：汇总逐章条目生成篇基本信息与篇主要内容

### 脚本
- `tools/split_chapters.py`：吞噬星空按章分割（CRLF，篇名+章名同行，最终篇第三十六章分上下合并）
- `tools/split_manghuang.py`：莽荒纪索引+分割（卷标记单独成行，章节`第X章`）
- `tools/split_feijian.py`：飞剑问道索引+分割（第一篇不带篇前缀，第二篇起带篇前缀，篇名可选，篇号递增过滤噪音）

### 项目规则
`吞噬星空大纲/项目规则.md`——所有agent必须遵守：
- 忠于原文，严禁臆造
- 存疑即查或问，绝不猜测（①回原文查 ②查state/相邻章 ③标记【存疑】中断回报）
- 章节不合并/不拆分，数据引用索引

### 格式标准（新格式，第9篇起统一）
```
# 第X篇 篇名 大纲
## 一、篇基本信息
## 二、篇主要内容（整体概括+核心主线+原文集标记对应）
## 三、逐章详细概括与类型标注
### 第X章 章名
**类型标签**：A、B
**内容**：忠于原文80-150字
**核心**：一句话
```
