## 三、技术架构

### 核心方案：单章读取 + 滚动状态档案 + 篇级子Agent并行
历史技术方案已移入 `docs/归档/`；当前工作流以各原作目录的索引、逐章大纲和 `_state/` 状态档案为准。

1. **按章分割**：脚本依据索引把原文切成单章文件，落盘可断点续传
2. **单章读取+滚动state.md**：每篇维护轻量状态档案（人物境界/伏笔/主线/近章摘要），替代全量原文上下文
3. **篇级子Agent并行**：每篇/卷开一个子agent独立处理，多篇同步生成
4. **篇级聚合**：汇总逐章条目生成篇基本信息与篇主要内容

### 脚本
- `tools/split_chapters.py`：吞噬星空按章分割
- `tools/split_manghuang.py`：莽荒纪索引+分割
- `tools/split_feijian.py`：飞剑问道索引+分割（处理篇名可选、篇号递增过滤噪音）

### 项目规则
当前规则入口为 `AGENTS.md`、`docs/当前接手卡.md`、`项目执行规则.md` 和 `原作分析规则.md`。本日志只记录历史，不重复维护读取预算、事实边界或 Git 流程。
