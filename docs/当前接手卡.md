# 当前接手卡

> 这是项目导航卡，不是质量报告；只用于新 Agent 快速定位，不代表本次任务已验证全库。

## 首次接手默认行为

用户首次提供仓库地址，或只说“了解/分析/看看项目情况”时，自动执行轻量概览，不要求用户指定模式：

- 先读 `AGENTS.md` 和本卡；
- 最多查看两层目录/文件名；用途不清时只读 README 开头摘要和少量配置/入口文件；
- 只概括用途、技术栈、主要目录和工作流程；无法确认的内容标记为待进一步读取；
- 概览后停止，等待用户指定下一步。

未经用户明确授权，禁止全库扫描、统计、正确性/完整性验证、重复/缺失检查、运行批量脚本、读取整部原文/全量附录/CSV/完整日志，也不得创建子 Agent 做这些事情。“仔细”“全面”“深入”不构成扩大授权。

- 项目：五部原作的逐章分析、大纲、state、汇总附录、特征数据和横向节奏研究。
- 正式作品：《吞噬星空》《莽荒纪》《飞剑问道》《沧元图》《雪鹰领主》。
- 接手顺序：先读 `AGENTS.md`，再读本卡；不要因为接手而打开完整 README、完整任务日志、整部原文或全量附录。
- 当前任务：以用户当前指令为准；本卡不保留临时任务，避免把旧待办误当成新任务。
- 当前阶段交接：state 档案补全、《莽荒纪》第二十二卷章节规范化、过期缺失说明清理和分割脚本路径可移植性已完成；剩余优化项以 `docs/任务日志.md` 顶部摘要为准，主要是章节异常的可选规范化、自动化验证体系、跨篇校对和旧格式迁移。
- 修改章节：只读目标规则条款、目标大纲、目标 state 和目标原文；需要时再增加相邻章节。
- 结构/数量/完整性核查：只有用户明确要求，或当前修改必须依赖时才做。
- 大文件查询：优先使用 `tools/query_material.py` 定点查询，不直接打开全量附录、CSV 或长原文。
- 状态读取：先取 state 的快速视图；只有跨篇连续性或冲突核查才读完整 state。
- 任务结束：只记录完成、未完成、存疑和下一步目标文件，不把整段过程留在聊天上下文中。
- Git：用户说“提交”即执行提交、推送工作分支、合并并推送 `main`，不二次确认；只有实际工具或权限失败才报告阻塞。

详细规则按触发条件读取：项目执行规则、原作分析规则、Git 交付规则均不是接手阶段必读资料。
