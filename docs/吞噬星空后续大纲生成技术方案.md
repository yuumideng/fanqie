## 二、格式标准（以最新格式为准）

现有大纲存在两代格式，**新篇统一采用第9、12篇的新格式**：

### 2.1 文件命名

`{两位序号}_{篇号全称}_{篇名}_大纲.md`，例：`13_第十三篇_兽神之道_大纲.md`。

### 2.2 文件骨架

```
# 第X篇 篇名 大纲

## 一、篇基本信息
- **篇名**：xxx
- **原文起止行**：xxxx-xxxx（据00_原文索引.md）
- **章节数**：xx章
- **字数**：xxx,xxx字
- **时间跨度**：……
- **空间跨度**：……

## 二、篇主要内容
（整体概括一段 + 按"xxx线"拆分核心主线 + 原文集标记对应）

## 三、逐章详细概括与类型标注

### 第1章 章名
**类型标签**：A、B
**内容**：忠于原文的概括（约80-150字）
**核心**：一句话点出本章核心
……（逐章至末章）
```

### 2.3 类型标签体系（从现有大纲归纳）

人物引入 | 设定展开 | 环境铺陈 | 矛盾展现 | 矛盾爆发 | 矛盾收束 | 战斗描写 | 爽点设置 | 转折高潮 | 情感展现 | 悟道 | 悬念设置 | 伏笔埋设 | 伏笔深化 | 设定揭示 | 情节推进 | 人物心理 | 氛围营造 | 装备获取 | 核心伏笔

每章选 1-3 个标签，用顿号 `、` 分隔。

### 2.4 章节编号

- 跟随原文章节标题（原文第10-19章写作"第一十章""第一十一章"等，第20章起为"第二十章"）。
- 大纲内 `### 第X章` 的 X 与原文编号一致，章名取原文章名。

## 三、核心架构：单章读取 + 滚动状态档案 + 篇级聚合

> 解决"整篇读原文导致上下文爆炸"与"单章独立解析丢失全局视角"的矛盾。

### 3.1 三种方案对比

| 维度 | 整篇读原文 | 单章独立解析（旧AI） | **单章+滚动状态（本方案）** |
|---|---|---|---|
| 上下文长度 | 爆炸（3000-14000行原文） | 极短（1章原文） | 短（1章原文+轻量状态档案） |
| 全局视角 | 有 | 无 | 有（状态档案） |
| 伏笔/主线判断 | 准 | 易失准 | 准 |
| 篇级信息生成 | 直接 | 难 | 篇末聚合生成 |
| 跨章境界/装备连续性 | 一致 | 易断裂 | 一致（状态档案追踪） |

**结论**：旧AI"控制上下文"方向对，但"完全不记上下文"会让跨章判断（伏笔属性、主线归属、境界延续）失准。优化方案用**轻量状态档案**替代全量原文上下文——既不堆原文，又不丢全局。

### 3.2 三段式架构

```
预处理：按章分割原文 → 单章文件（落盘，可复用/断点续传）
   ↓
逐章解析：[单章原文] + [滚动状态档案] → 本章大纲条目 + 更新状态档案
   ↓
篇级聚合：汇总全部章节条目 → 生成篇基本信息 + 篇主要内容 → 写入大纲文件
```

## 四、预处理：按章分割原文

### 4.1 为什么要物理分割

- 每次只读一章，上下文最短、最稳定；
- 落盘后可断点续传（解析到第20章中断，下次从第21章继续）；
- 可按章并行（配合状态档案）；
- 比用 offset/limit 每次重算行号更可靠。

### 4.2 分割方案（按篇组织，不平铺）

目录结构：
```
resources/吞噬星空_分章/
├── 第十三篇_兽神之道/
│   ├── 第1章_xxx.txt
│   ├── 第2章_xxx.txt
│   └── …（39个）
├── 第十四篇_域外战场/
│   └── …
└── …（17个子目录，共716个文件）
```

> 不采用1000+文件平铺，而是"按篇分目录"，定位清晰、与大纲篇对齐。

### 4.3 分割脚本逻辑

1. 读 `00_原文索引.md`，取每篇起止行；
2. 在篇范围内按章节标题正则切分（标题行顶格，正则：`^第[一二三四...十百零]+章\s+\S+`）；
3. 每章写入 `第X篇_篇名/第N章_章名.txt`，文件名取章名（去除非法字符）；
4. 输出分割校验：每篇切出的章数 == 索引章数，不一致则报错停下。

### 4.4 复用性

分割产物一次性生成，17篇共用；后续若做《莽荒纪》《飞剑问道》补全也可复用同脚本。

## 五、滚动状态档案（核心创新）

### 5.1 作用

替代"全量原文上下文"。解析每章时，只带入【本章原文 + 状态档案】，不带入历史原文。状态档案是**压缩后的结构化摘要**，体量约 1-2KB，远小于原文。

### 5.2 状态档案内容（每篇维护一份 `state.md`）

```
# 第十三篇 兽神之道 · 滚动状态档案

## 1. 人物境界线（当前值）
- 罗峰本尊：xxx境界，核心装备xxx，当前所在xxx
- 金角巨兽分身：xxx境界，核心能力xxx
- 魔杀族分身：xxx境界，核心能力xxx
- 其他关键人物：xxx（状态）

## 2. 伏笔追踪（已埋未解）
- [第X篇] xxx伏笔 —— 待回收
- [本篇第N章] xxx —— 待回收

## 3. 核心主线进度
- 主线A：xxx线 —— 当前推进到xxx阶段
- 主线B：xxx线 —— 当前推进到xxx阶段

## 4. 近章摘要队列（最近5-10章一句话）
- 第N-5章：xxx
- 第N-4章：xxx
- …
- 第N章：xxx（刚解析完）

## 5. 待确认/存疑
- xxx（解析中不确定的点，留待篇末复核）
```

### 5.3 更新时机

- 每解析完一章 → 更新"近章摘要队列""伏笔追踪""人物境界线"；
- 队列只保留最近5-10章，老的摘要已被吸收进"核心主线进度"，避免膨胀；
- 档案始终保持 1-2KB 量级。

### 5.4 与旧AI方案的关键差异

| | 旧AI（不记上下文） | 本方案（滚动状态档案） |
|---|---|---|
| 伏笔属性判断 | 单章看不出，易标错 | 伏笔追踪清单对照，标"埋设/深化/回收"准确 |
| 境界连续性 | 可能前后矛盾 | 状态档案记录当前境界，不跳级 |
| 篇主线归属 | 无依据 | 主线进度条提供归属判断 |
| 上下文增量 | 0 | +1-2KB（可忽略） |

## 六、单篇生成流程（优化版）

```
Step 0 [预处理] 跑分割脚本 → 得到本篇 N 个单章文件
Step 1 [初始化] 建本篇 state.md（境界线承接上篇结尾、伏笔追踪延续、主线进度清零）
Step 2 [逐章循环] for 每一章:
        读 [单章原文] + [当前 state.md]
        → 产出本章大纲条目（章名/类型标签/内容/核心）
        → 更新 state.md（摘要队列/伏笔/境界/主线）
        → 本章条目追加到篇级缓冲（不写最终文件）
Step 3 [篇级聚合] 基于全部章节条目 + state.md 最终态
        → 生成"篇基本信息"（时间/空间跨度从章节摘要归纳）
        → 生成"篇主要内容"（整体概括+核心主线+集标记对应）
Step 4 [写入] 组装 [篇基本信息 + 篇主要内容 + 逐章条目] → 写入大纲文件
Step 5 [校验] 章节数/起止行/忠于原文/格式/篇间衔接
```

**上下文构成（每章解析时）**：
- 格式标准说明（约0.5KB，固定）
- 当前 state.md（约1-2KB，滚动）
- 本章原文（约2-4KB，单章）
- 合计约 4-7KB，远低于整篇原文的 50-200KB。

## 七、执行模式：篇级子Agent并行（主推）

### 7.1 模式

- **每篇开一个子agent**，只负责该篇全部章节的"逐章解析 + 滚动状态档案 + 篇级聚合 + 写入大纲文件"。
- **主agent**负责：跑分割预处理 → 分发任务（注入统一模板）→ 监控 → 收集结果 → 跨篇一致性校对。
- 17篇可同步生成（受并发资源限制时分波启动），理论耗时≈最长单篇耗时，而非17篇串行累加。

### 7.2 为什么可行

- state.md 篇间独立，天然适合子agent隔离；
- 网文每篇开头会交代当前处境，子agent读本篇第一章即可获得起始境界/状态，**不强依赖前篇实时产出**——这正是17篇能同时启动的关键；
- 每个子agent上下文干净：仅装 [格式规范 + 范例篇 + 本篇索引 + 单章原文 + state.md]，互不干扰；
- 一个agent出错只影响一篇，天然隔离。

### 7.3 子Agent启动注入（保证17篇一致性）

每个子agent启动时统一注入：

1. **格式规范**（第二节全文）
2. **完整范例篇**：第12篇大纲作为格式样板
3. **本篇索引**：篇名/起止行/章节数/字数
4. **单章文件目录**：`resources/吞噬星空_分章/第X篇_篇名/`
5. **起始状态提示**（可选）：罗峰当前境界/关键未解伏笔——第13篇可从第12篇大纲结尾精确推算；14篇以后因前篇可能未完成，留空由原文首章自然获取

### 7.4 并发控制（分波启动）

全部17篇一波启动理论可行，但建议分3波以便验收与控制资源：

| 波次 | 篇目 | 数量 | 说明 |
|---|---|---:|---|
| 第1波 | 13-17 | 5 | 验证流程与跨篇一致性，主agent抽查2-3篇 |
| 第2波 | 18-23 | 6 | 第1波验收通过后启动 |
| 第3波 | 24-29（含最终篇） | 6 | 最终篇47万字单agent压力最大，单独关注 |

- 资源充足且第1波验收通过时，第2、3波可合并一波启动。
- 子agent采用异步团队模式（named member），spawn后不阻塞主agent，各自完成后回报。

### 7.5 跨篇一致性校对（全部完成后由主agent执行）

- **境界衔接**：通读各篇 state.md 结尾境界，确认无跳级/倒退；
- **伏笔跨篇回收**：未解伏笔是否在后续篇回收或合理延续；
- **格式统一**：骨架/标签/分隔符全局校对；
- **术语一致**：境界名/装备名/势力名全17篇统一。

### 7.6 断点续传

单章文件 + state.md 落盘 → 单篇agent中断可从下一章续；整篇失败可重新spawn该篇agent，不影响其他篇。

## 八、质量校验规则

每篇完成后执行：

1. **章节数校验**：大纲章节数 == 索引章节数 == 分割文件数。
2. **起止行校验**：首章对应起始行、末章对应终止行，能对上原文。
3. **忠于原文**：抽查3-5章，与单章原文比对，不得有臆造情节。
4. **格式一致性**：骨架、标签、分隔符符合第二节标准。
5. **篇间衔接**：本篇开篇 state.md 与上篇结尾 state.md 的人物/境界/悬念状态连贯。
6. **伏笔回收检查**：state.md 中"已埋未解"伏笔，在篇末确认是否回收或合理延续。

## 九、风险与应对

| 风险 | 应对 |
|---|---|
| 最终篇47万字、单篇超长 | 按章分割后天然化解，单章读取无压力；篇级聚合时按"集/阶段"分组汇总 |
| 章节编号特殊（第一十章等） | 分割脚本严格按原文标题正则，不自行改写编号 |
| 后期境界体系复杂 | state.md 的"人物境界线"持续追踪，保持术语一致 |
| state.md 漂移/遗漏更新 | 每章解析后强制更新；篇末校验伏笔清单是否清零/延续 |
| 分割脚本误切 | 分割后强制校验每篇章数==索引章数，不符即停 |
| 子agent输出格式漂移 | 统一注入格式规范+第12篇范例；全部完成后主agent全局校对 |
| 篇间境界衔接错位 | 起始状态由原文首章自然获取+篇末校对；必要时主agent手动修正衔接 |
