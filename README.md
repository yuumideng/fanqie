## resources 目录说明（已纳入私人仓库）

`resources/` 存放五部原作的原文 txt 与按章分割产物。五部原文由项目所有者本人创作，当前仓库为私人仓库，原文及分析产物均纳入版本控制，用于本项目的研究、整理和创作规划。

- `吞噬星空.txt` / `莽荒纪.txt` / `飞剑问道.txt` / `沧元图.txt` / `雪鹰领主.txt`：原始文本
- `吞噬星空_分章/` 等：由 `tools/split_*.py` 按章切分生成的单章文件

在新设备上延续工作时，直接同步或克隆整个私人仓库即可，无需另行准备原文 txt。

## 大纲生成技术方案

当前工作流以各原作目录的索引、逐章大纲和 `_state/` 状态档案为准；历史技术方案已移入 `docs/归档/`。核心架构：

1. **按章分割**：`tools/split_*.py` 依据 `00_原文索引.md` 把原文切成单章文件，落盘可断点续传；
2. **单章读取 + 滚动状态档案**：每篇/集/卷维护一个轻量 `state.md`（人物境界/伏笔追踪/主线进度/近章摘要），替代全量原文上下文，控制上下文长度；
3. **篇级子Agent并行**：每篇/集开一个子agent独立处理，多篇同步生成；
4. **篇级聚合**：汇总逐章条目生成篇基本信息与篇主要内容。

## 项目规则（按任务读取）

接手只读 `AGENTS.md` 和 `docs/当前接手卡.md`。具体任务再按四级读取规则定位资料；大文件优先使用 `tools/query_material.py`。项目执行边界、原作事实边界和 Git 交付细则分别见 `项目执行规则.md`、`原作分析规则.md`，不在 README 重复维护。

README 的目录、进度和完成度是导航信息，不是本次任务的质量验收结果；需要核查时只核查用户指定范围。

用户说“提交”即执行完整 Git 交付：提交、推送工作分支、合并并推送 `main`；除非认证、网络、权限或工具失败，不重复确认。

## 文档使用边界

当前工作只使用以下正式产物：

- 五部原作目录中的 `00_原文索引.md`、逐篇/逐卷大纲和 `_state/`；
- `五书全量逐章分析附录.md`、`五书全量逐章特征数据.csv`、`五书全量逐章结构节奏爽点对比分析.md`。
