## resources 目录说明（不入库）

`resources/` 存放小说原文 txt 与按章分割产物，因版权原因**不纳入 git 仓库**（见 `.gitignore`）：

- `吞噬星空.txt` / `莽荒纪.txt` / `飞剑问道.txt`：原始文本
- `吞噬星空_分章/`：由 `tools/split_chapters.py` 按章切分生成的单章文件（共1485个）

在新机器上延续工作时，需自行准备原文 txt 放入 `resources/`，再跑分割脚本生成分章文件。

## 大纲生成技术方案

详见 `docs/吞噬星空后续大纲生成技术方案.md`。核心架构：

1. **按章分割**：`tools/split_chapters.py` 依据 `00_原文索引.md` 把原文切成单章文件，落盘可断点续传；
2. **单章读取 + 滚动状态档案**：每篇维护一个轻量 `state.md`（人物境界/伏笔追踪/主线进度/近章摘要），替代全量原文上下文，控制上下文长度；
3. **篇级子Agent并行**：每篇开一个子agent独立处理，多篇同步生成；
4. **篇级聚合**：汇总逐章条目生成篇基本信息与篇主要内容。

## 项目规则（所有agent必读）

见 `吞噬星空大纲/项目规则.md`。核心铁律：

- **忠于原文，严禁臆造**；
- **存疑即查或问，绝不猜测**（①回原文查 ②查state/相邻章 ③仍无法确认则标记【存疑】中断回报）；
- 章节不合并/不拆分，数据引用索引。
