# 项目协作入口

这是一个大型小说创作与原作结构研究项目。任何 Codex、Agent 或人工协作者开始工作前，必须先阅读：

1. `项目执行规则.md`：全项目通用规则；
2. `README.md`：项目结构、当前进度和事实口径；
3. `CHANGELOG.md`：近期变更和交接记录；
4. 与任务直接相关的总纲、索引、state 和任务日志；
5. 若处理任何一部原作的逐章分析，再阅读 `原作分析规则.md`。

关键执行要求：

- 当前《界潮》执行版本是 22 卷、990 章；旧 36 卷、1620 章方案已废止，不得恢复；
- 原作事实与《界潮》原创设定必须严格分开；
- 不确定的事实必须查证或标记 `【存疑】`，不得猜测；
- 核心世界观、人物弧光、总纲、伏笔和结局必须保持单一主创视角，不能由多个 Agent 直接拼接；
- 长任务必须持续落盘并在阶段结束后做一致性校验；
- 用户要求“提交项目”时，必须新建 `codex/YYYYMMDD-XX` 分支，更新 `README.md` 与 `CHANGELOG.md`，执行 Git commit 和 Git push，并确认远程同步成功。

若当前任务需要改变主题、主角、结局、力量体系、卷数、章数或重新启用历史方案，先暂停并向用户确认。
