# 项目协作入口

这是项目协作入口，不是要求每次全量读取的项目说明。

## 默认读取

1. 先读本文件；
2. 再只读与当前任务直接相关的总纲、索引、state 或目标文件；
3. 按任务类型追加读取：
   - 项目结构/进度：读 `README.md` 的相关章节；
   - 《界潮》创作：读 `项目执行规则.md` 的相关章节和当前总纲；
   - 原作分析：读 `原作分析规则.md`、目标作品索引、目标篇章和 state；
   - 续接历史任务：只读 `docs/任务日志.md` 的相关区段；
   - 提交项目：只读 `CHANGELOG.md` 最新记录和 Git 交付规则。

默认不要读取完整 README、完整 CHANGELOG、完整任务日志、五书全量附录、无关作品目录或全部 resources。需要核对时，优先用目标文件、标题检索和局部行段读取。

核心创作规则、原作事实边界和 Git 交付流程分别以 `项目执行规则.md`、`原作分析规则.md` 和当前总纲为准。提交项目时，必须新建 `codex/YYYYMMDD-XX` 分支，更新 README 与 CHANGELOG，推送工作分支，成功后合并回 `main` 并推送 `main`。
