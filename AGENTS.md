# 项目协作入口

本文件只负责路由任务，不要求新 Agent 在接手阶段验证项目内容。

## 接手顺序

1. 先读本文件；
2. 再读 `docs/当前接手卡.md`；
3. 根据用户当前指令定位目标文件；
4. 只有实际修改或明确核查时，才读取原文、索引、state、目录和汇总数据。

项目概览阶段不检查大纲正确性、章节数量、文件结构完整性、历史问题或全库状态。README 和任务日志是导航/历史资料，不是接手阶段必读资料；其中的“完成”“已修复”等文字也只是路由信息，不能替代本次核查。

## 四级读取与默认预算

- **0级：导航**：只看本文件和接手卡，不打开业务资料。
- **1级：定位**：读取目标规则条款、目标文件局部和目标章节；默认不超过 1 个规则文件、1 个索引、1 个大纲/state、2 个原文分章文件。
- **2级：证据**：仅因事实不清或需要连续性，再增加相邻章节、相邻 state 或目标汇总条目。
- **3级：专项核查**：只有用户明确要求或任务必须依赖时，才检查目录、章节数量、分章完整性、全量附录、CSV 或更大范围原文。

扩大读取范围前先说明触发原因；没有矛盾就停在当前级别。大文件优先使用 `tools/query_material.py` 定点查询，不直接打开全量附录、CSV 或整部原文。

## 任务路由

- 项目概览：只读接手卡；用户明确要结构/进度时，再读 README 对应小节。
- 原作分析/大纲修订：读取原作规则的相关条款、目标索引、目标大纲、目标 state 和目标原文；不要默认读取整部作品。
- state/连续性维护：先取 state 快速视图；只有跨篇或冲突核查才取完整 state。
- 结构/数量/完整性审计：只在用户明确要求或本次修改确实依赖时执行。
- 续接历史任务：先读接手卡；确需追溯时才读任务日志相关历史区段。
- Git 交付：只在用户明确要求提交时读取项目执行规则第10节和 CHANGELOG 最新记录。

## 交付边界

不确定的原作事实必须回原文核对，仍无法确认时标记存疑并上报；发现问题不得自动扩大任务范围。子 Agent 只回报结论、文件、存疑和下一步，不复制整段原文或完整产物。

用户明确要求“提交项目”时，才执行新建 `codex/YYYYMMDD-XX` 分支、更新 README/CHANGELOG、提交、推送工作分支、合并回 `main` 并推送 `main` 的完整流程。未要求提交时，不主动执行 Git 提交或推送。
